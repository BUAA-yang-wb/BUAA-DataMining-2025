# 实验结果说明 (Results)

本目录存放了图像异常检测模型的训练过程记录和最终评估结果。

## 目录结构

每个类别（如 `hazelnut`, `zipper`）都有一个独立的子文件夹，其中包含：

1.  **`loss_curve.png` (训练损失曲线)**
    *   **含义**：展示了模型在训练过程中的 MSE Loss 变化情况。
    *   **解读**：曲线应呈现下降趋势并趋于平稳，表明模型成功学习了正常样本的特征重构。

2.  **`roc_curve.png` (ROC 曲线)**
    *   **含义**：展示了模型在不同阈值下的 True Positive Rate (TPR) vs False Positive Rate (FPR)。
    *   **解读**：曲线越靠近左上角，AUROC 分数越高（越接近 1.0），说明模型区分正常和异常样本的能力越强。

3.  **`visualization_result.png` (可视化检测结果)**
    *   **含义**：直观展示了模型对"正常样本"和"异常样本"的处理差异。
    *   **采样方式**：从测试集中选择**最后一个**正常样本和最后一个异常样本进行展示（结果是确定性的）。
    *   **包含三列**：
        *   **Input**: 原始输入图像。
        *   **Reconstruction**: 自动编码器重构后的图像。
        *   **Residual Error Map**: 残差热力图（红色/暖色区域表示高误差）。
    *   **解读**：
        *   对于 **Good Sample**，重构图应清晰，残差图应整体偏蓝（低误差）。
        *   对于 **Bad Sample**，重构图可能模糊或丢失缺陷细节，残差图在缺陷位置应出现明显的**红色高亮区域**，这正是模型检测到的"异常"。

4.  **`score_distribution.png` (异常分数分布直方图)**
    *   **含义**：展示了三组数据的异常分数分布：
        *   **Train (Good)** (绿色)：训练集中的正常样本。
        *   **Test (Good)** (蓝色)：测试集中的正常样本。
        *   **Test (Bad)** (红色)：测试集中的异常样本。
    *   **解读**：理想情况下，绿色和蓝色的分布应该重叠且位于左侧（低分区域），而红色的分布应该位于右侧（高分区域）。两组分布分离得越开，说明模型越容易区分正常和异常。

5.  **`all_results.csv` (详细检测结果)**
    *   **含义**：包含所有参与评估的图像（训练集+测试集）的详细检测数据。
    *   **字段**：
        *   `filename`: 图像文件名。
        *   `dataset`: 所属数据集 (train/test)。
        *   `true_label`: 真实标签 (good/bad)。
        *   `anomaly_score`: 模型计算出的异常分数 (MSE Loss)。

6.  **`autoencoder.pth` (模型权重)**
    *   **含义**：训练好的 PyTorch 模型参数文件，可用于后续加载和推理。

## 最终指标 (AUROC)

请参考终端输出的最终 AUROC 分数。
*   **AUROC = 0.5**: 相当于随机猜测。
*   **AUROC > 0.9**: 模型性能非常优秀。
